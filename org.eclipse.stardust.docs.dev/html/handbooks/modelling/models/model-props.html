Model Properties

Model properties can be viewed or modified in the model properties dialog. The following model properties can be accessed via the properties dialog:

  • General
    • Model OID - the identifier of the model. Please note that IDs containing hyphens, blanks or dots and IDs starting with digits are not supported.
    • Name
    • ID
    • Validity period (optionally defined by date entry specifying the beginning of the validity period)
    • Description - The description of the model is displayed as tooltip text in the Stardust Portal when you hover the mouse over the model name.
  • Configuration Variables - contains an editable list of all configuration variables defined in the model and their references. Chapter Using Configuration Variables describes this pane in detail.
  • Activity Criticality - contains JavaScript editor to specify and modify the activity criticality formula. For more information, please refer to the chapter Defining Activity Criticality.
  • Quality Assurance Codes - in this section you can add and modify Quality Assurance codes that can be used to track the reasons for Quality Assurance failures. Refer to section Quality Assurance Codes of chapter Quality Assurance on interactive Activities for details.
  • Simulation - helps you to specify simulation configurations (see chapter Simulation Configurations of the Simulation Guide.)
  • Effort Planning - for detailed information on how to set properties for effort planning please refer to the chapter Process Effort Calculation.
  • Language Bundles - settings for language bundles, see section Model of the Native Language Support chapter.

Editing Model Properties

You can edit and view model properties in the model properties dialog. To open this dialog, in the Outline tree view, right-click the model name and choose the option Properties
